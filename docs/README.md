# DBMS Architecture Knowledge Base

## Overview

This is the single Source of Truth and AI Knowledge Base representing the complete hierarchical DBMS architecture.

## Subsystems (Layer 2)

- [Data Storage & Access](./data-storage-access/README.md) - Handles physical layout, buffer caching, file I/O, page formatting, and disk/record representation.
- [Transaction & Concurrency Management](./transaction-concurrency-management/README.md) - Coordinates transaction lifecycles, concurrency control protocols, lock tables, and isolation guarantees.
- [Query Processing](./query-processing/README.md) - Parses, analyzes, optimizes, plans, and executes SQL statements, formatting results for output.
- [Database Objects](./database-objects/README.md) - Manages catalog metadata, schemas, tables, columns, indexes, constraints, views, and programmable objects.
- [Backup, Recovery & Logging](./backup-recovery-logging/README.md) - Manages transaction logging, write-ahead logs (WAL), checkpointing, backup operations, and database recovery.
- [Security & User Management](./security-user-management/README.md) - Controls user authentication, authorization policy, role hierarchies, and audits compliance.
- [Administration & Monitoring](./administration-monitoring/README.md) - Monitors system resources, configurations, database maintenance, import/export, and administrative tools.
- [Performance & Scalability](./performance-scalability/README.md) - Manages server memory, workloads, CPU cores, connection routing, clustering/distribution, and parallel execution.
